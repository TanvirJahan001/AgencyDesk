DATABASE SCHEMA
================================================================================

Collection: employee_documents/{docId}

Document Structure:
{
  id: string;                    // Auto-generated document ID
  employeeId: string;            // Employee UID
  employeeName: string;          // Snapshot of displayName at creation
  title: string;                 // Document title (required)
  category: string;              // One of 7 categories (required)
  description?: string;          // Optional notes
  fileUrl: string;               // External file URL (required)
  fileName: string;              // Display filename (required)
  fileType?: string;             // MIME type (optional)
  expiresAt?: string;            // ISO 8601 optional expiry date
  uploadedBy: string;            // Admin UID who created
  uploadedByName: string;        // Admin name snapshot
  status: string;                // "active" | "expired" | "archived"
  createdAt: string;             // ISO 8601 creation timestamp
  updatedAt: string;             // ISO 8601 update timestamp
}

Document Categories (7 options):
- contract (Employment Contract)
- id_document (ID Document)
- certificate (Certificate / License)
- tax_form (Tax Form)
- offer_letter (Offer Letter)
- policy (Policy Document)
- other (Other)

Status Values:
- "active" — Document is current and valid
- "expired" — Expiry date has passed
- "archived" — Document marked as old/inactive by admin

API ENDPOINT DETAILS
================================================================================

BASE: /api/documents

1. GET /api/documents
   Authentication: Required (any authenticated user)
   Authorization: 
     - Admin/CEO: Access all documents
     - Employee: Access only own documents
   Query Parameters:
     - employeeId (optional, admin/ceo only): Filter by employee
   Response: { success: true, data: EmployeeDocument[] }
   Errors: 401 (unauthorized), 403 (forbidden)

2. POST /api/documents
   Authentication: Required
   Authorization: Admin/CEO only
   Body:
     {
       employeeId: string (required),
       title: string (required),
       category: string (required, one of 7),
       fileUrl: string (required),
       fileName: string (required),
       description?: string,
       fileType?: string,
       expiresAt?: string (ISO 8601)
     }
   Validation:
     - All required fields present
     - Title/description/fileName under max length
     - Category in allowed list
     - Employee exists in database
   Auto-Computed:
     - employeeName: From employee record
     - uploadedBy: From session
     - uploadedByName: From session
     - status: "expired" if expiresAt < now, else "active"
     - createdAt/updatedAt: ISO 8601 now
   Response: { success: true, data: EmployeeDocument }
   Errors: 400 (validation), 401 (unauthorized), 403 (forbidden), 404 (employee not found), 500 (server)

3. PATCH /api/documents/{id}
   Authentication: Required
   Authorization: Admin/CEO only
   Parameters: id (document ID in URL)
   Body (all optional):
     {
       title?: string,
       category?: string,
       description?: string,
       fileUrl?: string,
       fileName?: string,
       fileType?: string,
       expiresAt?: string | null,
       status?: string
     }
   Validation:
     - Updated fields pass same validation as POST
     - Status in allowed values if provided
     - Category in allowed values if provided
   Auto-Computed:
     - If expiresAt provided and no status specified:
       • "expired" if expiresAt < now
       • "active" if expiresAt >= now
     - updatedAt: ISO 8601 now
   Response: { success: true, data: EmployeeDocument (merged) }
   Errors: 400 (validation), 401 (unauthorized), 403 (forbidden), 404 (document not found), 500 (server)

4. DELETE /api/documents/{id}
   Authentication: Required
   Authorization: Admin only
   Parameters: id (document ID in URL)
   Response: { success: true }
   Errors: 401 (unauthorized), 403 (forbidden), 404 (not found), 500 (server)
